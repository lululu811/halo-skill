# 让 AI 分析股票，你信它算出来的数字吗？

> 一个解决"AI 幻觉"的基本面分析框架：HALO。

![封面：AI 分析股票的数据可信问题](examples/wechat-images/halo-cover_001.jpg)

---

## 一个常见场景

你打开某个 AI 助手，输入：

> "分析一下贵州茅台，现在估值贵不贵？"

几秒钟后，AI 给你一段分析：

> "茅台 PE 约 18 倍，ROE 30%，毛利率 91%，处于近十年估值低位……"

看起来很有道理。但你忽然停下来：这些数字，是它从数据库查的，还是它"觉得应该是这样"的？

如果你追问"数据来源是什么"，很多 AI 会含糊其辞，或者直接告诉你"基于训练数据估算"。问题是，投资决策不能建立在"估计"上。

这就是当前 AI 股票分析最大的问题：**数据不可追溯**。

---

## AI 幻觉，在股票分析里尤其危险

大语言模型的本质是预测下一个词。它很擅长组织语言、梳理逻辑，但它并不天然"知道"今天的股价、上周的财报、昨天的主力资金流向。

更麻烦的是，当它不知道时，它可能会编一个"看起来合理"的数字。18 倍的 PE、30% 的 ROE、91% 的毛利率——这些数字对茅台来说确实不离谱，但如果是另一只你没那么熟悉的股票呢？你可能根本察觉不到它在胡说。

所以，让 AI 直接做股票分析，存在两个隐患：

1. **数字可能是错的**：模型没有实时数据源，只能依赖训练数据或推断。
2. **无法验证**：你拿不到原始数据，没法自己核对。

对普通投资者而言，这相当于在沙子上盖房子。

---

## HALO 的思路：数据层和分析层彻底分离

HALO（A股可信分析框架）换了一个思路：**不让 AI 碰数据，只让 AI 做判断。**

它的流程分成四步：

1. **Python 脚本从公开 API 拉取数据**：腾讯行情、东方财富、新浪财经、巨潮资讯等。
2. **生成 JSON 数据文件**：价格、PE、PB、市值、财报三表、财务比率、成长性、资金流……全部落盘。
3. **生成报告骨架**：所有数字由 Python 从 JSON 精确填充，AI 的分析槽位全部留空。
4. **AI 只填空**：在预留的 `{{AI_*}}` 槽位里写分析文字，**绝对不能修改已有的任何数字**。

也就是说，AI 看到的不是"它需要自己去算"的问题，而是一个"数字已经确定，你只需要解释和判断"的问题。

你看到报告里的每一个数字，都能从报告追溯到 JSON，从 JSON 追溯到 API。这是"可验证"的 AI 分析。

## 它长什么样？

以茅台（600519）为例，输入 `/halo 600519` 后，HALO 会生成一份 500 多行的报告，开头是这样的：

```
📋 执行摘要
├─ HALO评分：3.55/5.0 🟡强
├─ 成长性：6.5/10 🟡强
├─ 护城河：9.0/10 🟢极强
├─ 滞胀防御：8.5/10 🟢极强
└─ 综合评级：增持（目标价1400-1600元）
```

这些数字不是 AI 随口说的。它们来自：

- 腾讯行情 API：实时价、PE、PB、市值
- 新浪财报 API：利润表、资产负债表、现金流量表（各 8 期）
- 东方财富 API：行业、股本、公司信息、主力资金流向
- 自动生成：HALO 六维评分、成长性评分、财务比率

如果你打开 `data/600519.json`，能看到每一个数字的原始值和来源。AI 只负责解释"为什么护城河极强"、"为什么成长性只是中等"，以及给出投资建议。

再以 TCL 科技（000100）为例，这是一份完整的 640 行报告，包含了第十二章的产业链分析。HALO 会根据公司所属行业，自动生成产业链定位、稀缺性评级、证据链和风险点。这不是 AI 的"感觉"，而是基于行业匹配和财务数据的启发式分析。

## 11 个维度，而不是只看 PE

HALO 不做技术面信号，也不给具体买卖点。它回答的问题是：**这家公司值不值得深入研究？**

为了回答这个问题，它用了 11 个维度：

| 维度 | 权重 | 衡量什么 |
|:-----|:----:|:---------|
| HALO 六维 | 15% | 重资产属性，按行业调整 |
| 成长性 | 15% | 营收/利润增速与质量 |
| 护城河 | 15% | 技术壁垒、客户粘性、竞争格局 |
| 滞胀防御 | 10% | 实物资产、定价权、现金流 |
| ESG | 10% | 环境、社会、治理 |
| 管理层 | 10% | 战略、资本配置、诚信 |
| 资金面 | 5% | 股东、北向、主力、融资 |
| 估值 | 10% | PE/PB/DCF 多维度 |
| 风险 | -10% | 7 类风险矩阵 |

这里有一个关键设计：**行业公平评分**。

银行、电力、白酒、软件，它们的资产结构天差地别。用同一套标准打分，对谁都不公平。HALO 把公司分成重资产型、混合型、轻资产型三类，分别设置评分阈值。比如同样是"固定资产密集度"，银行 100% 是正常，软件公司 30% 就已经很重了。

## 不局限于某个 AI 平台

HALO 最初是在 Claude Code 里跑通的，但它不是一个 Claude 专属工具。

它的核心是几个 Python 脚本 + Markdown 报告模板。只要你的 AI Agent 支持 Skill 机制，能够调用本地脚本、读取 Markdown、填空式生成内容，HALO 就能跑起来。

也就是说，国产 Agent、企业私有部署的 Agent、甚至你自己搭的本地 Agent，理论上都可以接入 HALO。数据层是 Python 锁死的，分析层是通用的 AI 填空逻辑——它不绑定任何一家模型厂商。

## 它适合谁？不适合谁？

**适合：**

- 想做基本面研究，但不想手动扒财报的投资者
- 对 AI 分析有疑虑，希望每个数字都可追溯的人
- 喜欢"框架化思考"，而不是听 AI 给买卖建议的人

**不适合：**

- 想找"明天买哪只"短线信号的人
- 完全不想看任何数字，只想听结论的人
- 希望 AI 替自己做投资决策的人

HALO 给自己的定位很清楚：**它是一套分析框架，不是投资顾问。** 它帮你把公司拆开、把数据摆出来、把逻辑理清楚，但最后买不买，还是你自己决定。

## 最后说两句

AI 投资分析最大的瓶颈，不是 AI 不够聪明，而是 AI 不够可信。

HALO 没有解决这个问题的一切，但它做了一个很基础、也很重要的事：**把"数据"和"判断"分开，让数据可追溯，让判断有根基。**

如果你也对"AI 分析股票"这件事半信半疑，不妨用一个你能验证数字的框架开始。
